# Slice 7c — Containers & Image Supply-Chain Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make containers + image supply-chain first-class via a **conditional, service-scoped** standard, a fail-closed conformance check, and one fully-worked reference profile (`typescript-node`) — without forcing containerization on non-service stacks and without adding a universal CI gate.

**Architecture:** Contract (stack-neutral standard in `DEVELOPMENT-STANDARDS.md` §14/§13 + `DEVELOPMENT-PROCESS.md` §9) → Reference (`profiles/typescript-node/`: Dockerfile, compose, devcontainer, extended `ci.yml`, k8s + Helm) → Conformance (`conformance/container-supply-chain.sh`, conditional-on-Dockerfile). The image SBOM uses **Syft (`anchore/sbom-action`, CycloneDX file)** and provenance binds the **image digest** via `actions/attest-build-provenance` on the existing push-only `provenance` job. The 8 universal `ci-gates.sh` ids are left intact — the regression lock that keeps this MINOR (→ 2.15.0).

**Tech Stack:** Docker (multi-stage, distroless non-root), Docker Compose, devcontainer, GitHub Actions (GHCR, OIDC), Kubernetes + Helm, POSIX `sh` conformance, Markdown.

**Spec:** `docs/superpowers/specs/2026-06-08-slice7c-containers-design.md` (approved). **Branch:** `feature/slice-7c-containers` (already created; spec committed).

## Governance & conventions (read before any task)

- Feature branch → PR → **human ratification**. Do **not** merge; agents never self-merge. Governing-doc changes (STANDARDS/PROCESS) get the Security-Owner lens.
- **Guard hazard:** the live `.claude/` PreToolUse guard scans **Bash command text** (not file contents written via the Write tool) and blocks literal destructive command strings (DB drops, `flushall`, recursive deletes, etc.). Keep all commit messages and shell commands free of such literals. Writing a `compose.yaml`/manifest via Write is fine; only Bash text is scanned.
- **The regression lock:** after editing `profiles/typescript-node/ci.yml`, `sh conformance/ci-gates.sh profiles/typescript-node/ci.yml` MUST still pass — the 8 universal gate-ids (`gate-lint gate-type-check gate-test gate-build gate-secret-scan gate-dep-scan gate-sbom gate-provenance`) must remain intact and unmodified. The two new ids (`gate-image-sbom`, `gate-image-provenance`) are **additive**, never substitutes.
- **Conditionality is the whole point:** non-service profiles (terraform, library/batch) get **no** container files; `container-supply-chain.sh` skips any profile without a `Dockerfile` (prints `N/A`, never fails it).
- Reference files are **inert in the kit** (GitHub only runs workflows under `.github/workflows/`; the Dockerfile/compose live under `profiles/typescript-node/`). They are COPY-&-ADAPT templates. Mark them as such in a header comment, matching the existing `ci.yml` header style.

```dockerfile
# syntax=docker/dockerfile:1
# Reference container image for the TypeScript/Node profile.
# COPY & ADAPT: copy to your project root as `Dockerfile`. INERT here in the kit.
# Satisfies DEVELOPMENT-STANDARDS.md §14 (container image supply-chain):
# multi-stage, non-root, minimal base, healthcheck.

# ---- builder: full toolchain, never shipped ----
FROM node:24-bookworm-slim AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build            # produces dist/
RUN npm prune --omit=dev     # keep only production deps for the runtime copy

# ---- runtime: distroless, non-root by construction ----
# Node 24 distroless ships on debian13. If your app has native modules, match the
# builder OS to Debian 13 (trixie) so prebuilt binaries are ABI-compatible.
FROM gcr.io/distroless/nodejs24-debian13:nonroot AS runtime
WORKDIR /app
ENV NODE_ENV=production
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/package.json ./package.json
# Numeric UID (distroless 'nonroot' = 65532): stays non-root even on :debug/:latest tags
# where the 'nonroot' name isn't defined, and matches the k8s securityContext runAsUser.
USER 65532
EXPOSE 3000
# Distroless has no shell; use exec form. dist/healthcheck.js exits non-zero on failure.
HEALTHCHECK --interval=30s --timeout=3s --start-period=10s --retries=3 \
  CMD ["node", "dist/healthcheck.js"]
CMD ["dist/server.js"]
```

- [ ] **Step 1: Replace the file with the extended pipeline**

Overwrite `profiles/typescript-node/ci.yml` with EXACTLY:

```yaml
# Reference CI pipeline for the TypeScript/Node profile.
# COPY & ADAPT: copy this file to your project's .github/workflows/ci.yml.
# It is INERT here in the kit repo (GitHub only runs workflows under .github/workflows/).
# Each quality gate carries a standardized `id: gate-*` that conformance/ci-gates.sh asserts.
# Satisfies DEVELOPMENT-STANDARDS.md §14 (CI/CD Pipeline) incl. the conditional
# container image supply-chain (gate-image-sbom + gate-image-provenance).
#
# HARDENING (do before production):
#  - Pin every `uses:` to a full 40-char commit SHA (e.g. actions/checkout@<sha>  # v4.x).
#  - Pin tool installs to exact versions.
#  - OIDC is least-privilege here: id-token:write / packages:write live only on the
#    push-only provenance jobs, so PR-triggered steps cannot mint a token or push an
#    image. Your cloud trust policy MUST restrict `sub` to
#    `repo:<org>/<repo>:ref:refs/heads/main` (never `pull_request`).
name: CI

on:
  pull_request:
  push:
    branches: [main]

permissions:
  contents: read

jobs:
  ci:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0   # full history so the secret scan sees all commits

      - uses: actions/setup-node@v4
        with:
          node-version: '24'
          cache: npm

      - name: Install dependencies
        id: gate-install
        run: npm ci

      - name: Lint
        id: gate-lint
        run: npm run lint

      - name: Type-check
        id: gate-type-check
        run: npm run type-check

      - name: Test + coverage (>=80%)
        id: gate-test
        run: npm run test:coverage   # vitest --coverage; thresholds enforced in vitest.config.ts

      - name: Build
        id: gate-build
        run: npm run build

      - name: Secret scan
        id: gate-secret-scan
        uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          # Organization-owned repos additionally require a gitleaks license:
          # GITLEAKS_LICENSE: ${{ secrets.GITLEAKS_LICENSE }}

      - name: Dependency vulnerability scan
        id: gate-dep-scan
        run: npm audit --audit-level=high

      - name: Generate SBOM (CycloneDX)
        id: gate-sbom
        run: npx --yes @cyclonedx/cyclonedx-npm --output-format JSON --output-file sbom.json

      - name: Upload SBOM
        uses: actions/upload-artifact@v4
        with:
          name: sbom
          path: sbom.json

      # --- conditional container image supply-chain (service profile) ---
      # Builds the image and scans it on EVERY PR (real scan-before-merge); no push here.
      - name: Build container image (local, for SBOM scan)
        run: docker build -t ts-node-app:ci .

      - name: Generate image SBOM (Syft, CycloneDX)
        id: gate-image-sbom
        uses: anchore/sbom-action@v0
        with:
          image: ts-node-app:ci
          format: cyclonedx-json
          output-file: image-sbom.json

      - name: Upload image SBOM
        uses: actions/upload-artifact@v4
        with:
          name: image-sbom
          path: image-sbom.json

      - name: Upload build artifact (for provenance)
        if: github.event_name == 'push' && github.ref == 'refs/heads/main'
        uses: actions/upload-artifact@v4
        with:
          name: build-artifact
          path: dist/**           # the real build output (npm run build)
          if-no-files-found: error

  provenance:
    needs: ci
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    permissions:
      contents: read
      id-token: write       # scoped to this push-only job (build-provenance attestation)
      attestations: write
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: build-artifact
          path: build-artifact
      - name: Attest build provenance
        id: gate-provenance
        uses: actions/attest-build-provenance@v1
        with:
          subject-path: build-artifact/**

  image-provenance:
    needs: ci
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write       # push image to GHCR (scoped to this push-only job)
      id-token: write       # OIDC for digest-bound attestation
      attestations: write
    steps:
      - uses: actions/checkout@v4
      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - name: Build and push image
        id: build-push
        uses: docker/build-push-action@v6
        with:
          context: .
          push: true
          tags: ghcr.io/${{ github.repository }}:${{ github.sha }}
      - name: Attest image provenance (digest-bound)
        id: gate-image-provenance
        uses: actions/attest-build-provenance@v1
        with:
          subject-name: ghcr.io/${{ github.repository }}
          subject-digest: ${{ steps.build-push.outputs.digest }}
          push-to-registry: true
```

- [ ] **Step 1: Write the script**

Create `conformance/container-supply-chain.sh`:

```sh
#!/bin/sh
# container-supply-chain.sh — conditional, fail-closed image supply-chain check.
#
# For each profile directory: IF a Dockerfile is present, the image MUST be
# multi-stage and run as a non-root final USER, AND its sibling ci.yml MUST declare
# gate-image-sbom + gate-image-provenance with a digest-bound attestation. Profiles
# WITHOUT a Dockerfile are N/A (skipped), never failed — this keeps the standard
# conditional (DEVELOPMENT-STANDARDS.md §14) so non-service stacks aren't forced to
# containerize. Checks contract identifiers, not stack tools — stack-neutral.
# Run at the Review gate (DEVELOPMENT-PROCESS.md §7).
set -eu

ROOT="${1:-profiles}"
fail=0
checked=0

for dir in "$ROOT"/*/; do
  df="${dir}Dockerfile"
  name="${dir%/}"
  if [ ! -f "$df" ]; then
    echo "N/A (no Dockerfile): $name"
    continue
  fi
  checked=$((checked + 1))
  ci="${dir}ci.yml"
  pfail=0

  # 1. multi-stage: at least two FROM stages
  froms=$(grep -cE '^[[:space:]]*FROM[[:space:]]' "$df" || true)
  if [ "$froms" -lt 2 ]; then
    echo "FAIL $name: Dockerfile not multi-stage (need >=2 FROM, found $froms)"
    pfail=1
  fi

  # 2. non-root: the FINAL USER directive governs the runtime; it must not be root/0/empty
  last_user=$(grep -E '^[[:space:]]*USER[[:space:]]' "$df" | tail -1 | awk '{print $2}')
  case "$last_user" in
    "" | root | 0)
      echo "FAIL $name: final USER is root or unset ('$last_user')"
      pfail=1
      ;;
  esac

  # 3. sibling ci.yml declares both conditional image gate-ids
  if [ ! -f "$ci" ]; then
    echo "FAIL $name: Dockerfile present but no sibling ci.yml"
    pfail=1
  else
    for id in gate-image-sbom gate-image-provenance; do
      grep -qE "id:[[:space:]]*$id" "$ci" || { echo "FAIL $name: ci.yml missing $id"; pfail=1; }
    done
    # 4. provenance binds the image digest (not only a file path)
    grep -qE 'subject-digest|push-to-registry' "$ci" || {
      echo "FAIL $name: image provenance not digest-bound (need subject-digest or push-to-registry)"
      pfail=1
    }
  fi

  if [ "$pfail" -eq 0 ]; then
    echo "OK $name: container supply-chain present (multi-stage, non-root, image SBOM + digest provenance)"
  else
    fail=1
  fi
done

if [ "$fail" -ne 0 ]; then
  echo "container-supply-chain: FAIL" >&2
  exit 1
fi
echo "container-supply-chain: OK ($checked profile(s) with a Dockerfile checked; others N/A)"
```
