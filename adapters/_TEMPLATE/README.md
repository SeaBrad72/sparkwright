# adapters/__HARNESS__/

BYO adapter skeleton — generated by `sh scripts/new-adapter.sh __HARNESS__`.

This adapter ships **floor-only**: every dimension is at the Kit-enforced floor and `mcp-gate` is `n-a`. A freshly-generated adapter conforms immediately — `sh conformance/harness-adapter.sh adapters/__HARNESS__` exits 0 — because the floor holds through the universal governance layer (the git hook + CI backstop) without any harness-native interception.

## What this adapter declares

| Dimension | Level | Enforced by |
|-----------|-------|-------------|
| context-binding | floor | `AGENTS.md` present + routes to canonical docs |
| command-guard | floor | `hooks/pre-push` + `scripts/kit-guard` + `conformance/agent-boundary.sh` |
| history-protection | floor | `hooks/pre-push` (force-push / push-to-main guard) |
| review-roles | floor | `conformance/agent-boundary.sh` + `conformance/branch-protection.sh` |
| mcp-gate | n-a | No MCP surface declared for this harness |

## How to customize

1. **Set `controlPlanePaths`** — replace or extend the defaults with the actual control-plane paths for the `__HARNESS__` namespace (e.g. its config file, rules directory, or settings path).
2. **Upgrade a dimension to `"native"`** — if the harness supports inline pre-exec interception, native subagents, or an MCP gate, change the `"level"` to `"native"` and add a `"proof"` (`check` and/or `files`) so `harness-adapter.sh` can verify the claim rather than reject it as lying-native.
3. **Validate** — run `sh conformance/harness-adapter.sh adapters/__HARNESS__` after each change.

## Floor vs. native

The floor is the equal-enforcement guarantee: every harness provides it, no exceptions. Native is additive — it tightens enforcement when the harness supports inline command interception. A `native` claim without a passing proof is caught by the lying-native guard; a `floor` claim with no native capability is fully covered by the universal layer.
